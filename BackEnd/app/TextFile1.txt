*Mục đích: Tiếp cận chức năng của ứng dụng dưới dạng API RESTful, xử lý yêu cầu và phản hồi HTTP.

*Các thành phần chính:

-Controllers (Bộ điều khiển): Nhận yêu cầu HTTP và gọi các dịch vụ ứng dụng.
-DTOs (Đối tượng chuyển đổi dữ liệu): Sử dụng để mô hình hóa dữ liệu trao đổi giữa API và các client.
-Middleware: Xử lý các quan tâm chung như xác thực và phân quyền.
*Nguyên tắc chính:

Thiết kế trước API: API được thiết kế độc lập với UI, thúc đẩy khả năng tái sử dụng.
Hypermedia như Engine of Application State (HATEOAS): Phản hồi API có thể bao gồm liên kết đến các tài nguyên liên quan để có thể khám phá.
